ocgt-monitor

OpenCode Go 套餐额度监控工具 — 一步一步教你用

PowerShell（蓝色窗口）用户注意： 运行命令时要在前面加 .\，例如 .\ocgt-monitor quota。
如果是 cmd（黑色窗口），直接输入 ocgt-monitor quota 就行。
这个工具是做什么的？
ocgt-monitor 大部分功能与 ocgt 无关，它直接调用 OpenCode Go 的接口查询额度。
只有 history（查看历史消耗）需要读取 ocgt 产生的日志文件。

先搞懂几个词

终端 / 命令行 / cmd — 那个黑底白字（或蓝底白字）的窗口。按 Win 输入 cmd 回车就能打开。
为什么有 exe 还要敲命令？ ocgt-monitor 有 11 种功能（查额度、看历史、启动网页…），需要在终端里输入不同的命令来告诉它你想做什么。
Cookie — 网站上登录后浏览器存的一个"通行证"，复制出来交给 ocgt-monitor 去查额度。

第一步：打开终端

Step 1

按 Win 键 → 输入 cmd → 回车。
你会看到一个黑色窗口，上面有 C:\Users\你的用户名> 这样的文字。

第二步：进入程序所在文件夹

Step 2

在终端里输入下面这行命令，按回车：

C:\Users\你的用户名> cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor

回车后提示符变成 C:\Users\...\ocgt-monitor> 就对了。

如果你把 exe 放在了别的位置，这里改成你实际的路径。

第三步：获取 Cookie 和 Workspace ID

Step 3
  1. 打开 Chrome 浏览器，登录 opencode.ai
  2. 按 F12 打开开发者工具
  3. 点顶部 Application（应用）
  4. 左侧找到 Cookies → https://opencode.ai
  5. 随便点一个 Name，按 Ctrl+A 全选，Ctrl+C 复制
  6. 粘贴到记事本里存着

Workspace ID： 浏览器地址栏找到 /workspace/wrk_xxxxxxxxxxxx/usage，把 wrk_ 开头那串复制下来。

第四步：运行配置向导

Step 4

在终端里输入：

C:\Users\...\ocgt-monitor> .\ocgt-monitor config init

然后跟着提示回答：

  1. 按回车继续
  2. 粘贴 Cookie，按回车
  3. 输入 Workspace ID，按回车
  4. DeepSeek API Key 直接回车跳过（除非你有）

看到"配置已保存"就成功了。

配置只需要做一次。以后直接查额度就行。

第五步：查套餐额度

Step 5
C:\Users\...\ocgt-monitor> .\ocgt-monitor quota

会显示 Rolling / Weekly / Monthly 三条进度条，告诉你套餐用了多少。

如果报错，Cookie 可能过期了。重新复制后运行 .\ocgt-monitor config init 更新。

其他常用命令

.\ocgt-monitor balance查 DeepSeek 余额
.\ocgt-monitor history看最近 7 天 Token 消耗
.\ocgt-monitor watch持续监控（60秒刷新），Ctrl+C 退出
.\ocgt-monitor serve启动 Web 面板，浏览器打开 http://127.0.0.1:8788

分享给朋友

运行下面这个命令清空你的配置，然后把 exe 发过去就行：
.\ocgt-monitor config delete default

常见问题

Q: 提示"不是内部或外部命令"？
A: 没有先 cd 进文件夹。先运行 cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor。
Q: PowerShell 报错"无法识别"？
A: 要在 exe 前面加 .\，写成 .\ocgt-monitor quota。
Q: 配置好了但查 quota 报错？
A: Cookie 过期。重新登录 opencode.ai 复制，运行 .\ocgt-monitor config init 更新。
Q: history 无数据？
A: 没用 ocgt 的话就没有历史日志。但 quota 和 balance 不受影响。
