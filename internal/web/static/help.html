ocgt-monitor

OpenCode Go plan quota & token usage dashboard — CLI, web, and desktop sidebar

!
PowerShell users: prefix commands with .\ — e.g. .\ocgt-monitor quota.
CMD users: just type ocgt-monitor quota directly.

Overview

ocgt-monitor is a standalone tool for monitoring your OpenCode Go plan usage. It queries the OpenCode Go RPC API directly — no dependency on ocgt proxy except for the history command, which reads ocgt's JSONL log files.

CLI Web Dashboard Desktop Sidebar

Quick Start

1

Open a terminal

Press Win, type cmd, hit Enter. Then navigate to the project:

C:\Users\...> cd C:\Users\14050\Desktop\opencodego_tokens\ocgt-monitor
2

Gather credentials

Log into opencode.ai, press F12 → Application → Cookies → copy the full cookie string. Also note your Workspace ID (wrk_xxx) from the URL.

3

Run setup wizard

C:\Users\...\ocgt-monitor> .\ocgt-monitor config init

Follow the prompts: paste cookie, enter workspace ID, skip DeepSeek key if unused.

4

Check your quota

C:\Users\...\ocgt-monitor> .\ocgt-monitor quota

Shows Rolling / Weekly / Monthly usage with progress bars and reset timers.

Multiple Accounts

Config is stored at C:\Users\<you>\.ocgt-monitor\config.json — each Windows user gets their own isolated config.

.\ocgt-monitor config add work       # add another account
.\ocgt-monitor config use work       # switch to it
.\ocgt-monitor config list           # list all accounts

Sharing the Binary

!
Before sending ocgt-monitor.exe to someone else, clear your config:
.\ocgt-monitor config delete default
The recipient runs config init to set up their own credentials.

Environment Variables

Env vars take precedence over the config file — useful for quick testing:

:: temporary (current terminal)
set OPENCODE_GO_AUTH_COOKIE=session=xxx;.....
set OPENCODE_GO_WORKSPACE_ID=wrk_xxxxxxxxxxxx

Architecture

Data sources
OpenCode Go RPC (_server)    ───→  quota (plan usage)
DeepSeek API (/user/balance) ───→  balance (pay-as-you-go)
ocgt JSONL logs (~/.ocgt/)   ───→  history (token consumption)

UI modes
CLI   →  terminal commands
Web   →  browser at http://127.0.0.1:8788
Panel →  desktop sidebar (--sidebar)

FAQ

No credentials configured?
Run config init to set them up.
API call failed?
Cookie expired. Re-copy from opencode.ai and run config init.
History empty?
Only available if ocgt is running (reads its JSONL logs). Quota and balance work independently.
Need to update a field?
Re-run config init — existing values shown as defaults; press Enter to keep.
Sidebar doesn't appear?
Ensure WebView2 Runtime is installed (built into Win11). Build with CGO_ENABLED=1 and GCC in PATH.
Go or GCC not installed?
The binary is pre-built — you only need Go/GCC if you want to rebuild from source.
